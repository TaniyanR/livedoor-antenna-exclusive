# livedoorブログ側の表示設定

管理ツールは、1回の投稿本文へ最大50件の画像カードを出力します。記事ページではPC横5列・最大10段となり、各カードから元サイトの記事へ移動します。

## 管理ツール側で自動的に行うこと

- 各RSSサイトから、できるだけ同じ件数の記事を選択
- 記事が少ないサイトの不足分を、残りのサイトへ公平に再配分
- 同じサイトの記事が連続しにくいようにランダム配置
- 1投稿あたり最大50件
- 代表記事は、その回に選んだ記事からランダム選択
- 直前の代表サイトを可能な限り避け、特定サイトへの偏りを抑制
- 記事ページはPC横5列
- 画像は縦横比を維持して表示
- コメント欄を投稿本文内のCSSで非表示

## トップページの設定

トップページは、livedoor Blog側で次の設定にしてください。

1. livedoor Blog管理画面の「ブログ設定」を開きます。
2. トップページの記事表示件数を **40件** にします。
3. ページ送りを有効にします。
4. PC用デザインは、記事を **横5列・縦8段** で並べるデザインにします。
5. 各トップカードのリンク先は、投稿されたlivedoor記事ページにします。
6. トップカードには、投稿タイトルと記事本文の代表画像を表示します。

41件目以降は、livedoor Blogのページ送りで次ページへ表示されます。

## 記事ページ

- PCでは横5列、最大50件です。
- 各カードはRSS元サイトの記事へ直接リンクします。
- スマートフォンでは画面幅に応じて2列または1列になります。
- コメント欄は非表示になります。

## 現在の崩れを直す手順

1. livedoor Blogの「ブログ設定」→「デザイン／ブログパーツ設定」→「PC」を開きます。
2. 現在のカスタムデザインを保存してバックアップします。
3. HTML内に表示されている ` ```html ` と、末尾の ` ``` ` を削除します。
4. トップページで記事本文を表示しない独自テンプレートを使用している場合は、いったんlivedoor標準デザインへ戻します。
5. 新しい投稿を1件テストし、画像カードが表示されることを確認します。

## 注意

- 管理ツールはAtomPub APIで記事を投稿しますが、livedoor Blog側のトップページ表示件数やテンプレートをAPIから変更することはできません。
- トップページの40件表示と5列化は、livedoor Blog管理画面での設定が必要です。
- 修正前に投稿済みの記事本文は自動では書き換わりません。修正後に作成した新しい投稿から新レイアウトが適用されます。
